# 未来方向

这里不是排期承诺，更像一张公开草稿：我们把现在想做、也希望社区一起试的方向放在这里。做出来之前，它都只是方向。

## 可替换头部

YareLampGo 现在首先是台灯，但我们不想把它锁死在“灯”这个形态里。后续更理想的结构是磁吸或卡扣式头部组件，换起来简单，也方便社区自己做小模块。

可能的头部方向：

- 灯头：照明、补光、表情灯。
- 手机支架：把机械臂变成会动的桌面拍摄支架。
- 小音箱：让语音反馈和动作放在同一个桌面设备上。
- 吹风或气流模块：这类会涉及功率、温控和安全保护，只适合在额外安全设计完成后再尝试。

## 算法扩展

这个仓库现在没有内置 VLA、RL 或大规模训练流程。我们先把硬件、动作、安全边界、Web 控制和 Agent 接口打通，让设备能被普通开发者跑起来。

但这不代表我们不关心算法。相反，我们希望以后有人把这些方向接进来：

- 数据采集和动作数据集整理。
- 模仿学习、策略学习和 RL。
- VLA / 视觉语言动作模型。
- 更好的视觉感知、目标跟踪和桌面场景理解。

如果你想贡献这类能力，最好的切入点不是重写整套系统，而是先接入现有的动作、技能、Web API 或 OpenClaw 接口。

## 桌面场景

我们很期待看到更生活化的玩法，比如：

- 桌面拍摄助手。
- 直播或视频里的反应道具。
- 跟随音乐或语音情绪做动作。
- 带确认机制的 Agent 硬件执行器。
- 适合普通用户自己组合的动作和 skill 场景。

小项目先好玩、好改、好复现，才有机会长出更严肃的东西。

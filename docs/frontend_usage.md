# Lampgo 前端能力总览与演示剧本

> 文档定位：**给老板看的产品 Demo 文档**。逐个模块讲清"这块界面做了什么、为什么重要"，并配套 6 个端到端演示 Case，帮助快速建立对 Lampgo Web 控制台的整体认知。
>
> 演示地址：`http://127.0.0.1:8420`（守护进程启动后自动开放）
> 启动命令：`uv run lampgo run --web`

---

## 0 · 一张图认识界面

![整体布局](img/00_overview.png)

界面由三层组成：

- **左侧 Sidebar** — 品牌区 + 连接状态徽标 + 历史会话搜索 + 6 个一级视图入口 + 历史会话列表。
- **顶部 Topbar** — 4 个设备胶囊（关节角度 / 摄像头 / 麦克风 / LED）实时反映硬件在线状态，ESP32 在线时多一个音量滑杆；右上角是醒目的 **安全急停** 按钮。
- **中部主视图** — 根据左侧导航切换，覆盖聊天、通话、技能、OpenClaw、日志、配置 6 大场景。

> 设计原则：所有关键状态（连接 / 设备 / 在跑任务 / 配置覆盖来源）都在第一屏可见；危险操作（急停 / 删除设备 / 恢复默认）显式标红。

## 1 · 模块能力清单

### 1.1 新建聊天（Chat）

![聊天视图](img/10_chat.png)

| 能力 | 说明 |
| --- | --- |
| 流式对话 | 文字 / 语音输入，回复逐 token 渲染，支持「停止」中断长任务 |
| 工具调用气泡 | 每次 LLM 调用工具（动作 / LED / 摄像头抓帧 / 联网搜索）都会展开成可折叠卡片，参数、返回值、耗时一目了然 |
| 历史会话 | 自动保存于 `~/.lampgo/sessions.json`，左侧侧边栏可搜索 / 切换 / 删除 |
| 首屏 Hint Chip | 4 个一键提示词（点头 / 切换表情 / 分析现场 / 回到安全位），降低首次上手门槛 |
| 语音输入 | 按住麦克风按钮录音 → STT → 注入输入框；可选麦克风设备下拉切换 |

### 1.2 通话（Realtime Call）

![通话视图](img/20_call.png)

| 能力 | 说明 |
| --- | --- |
| 一键发起实时语音对话 | 走 LiveKit + 火山引擎 ASR/TTS，全双工，低延迟 |
| ASR / 助手回复实时显示 | 用户语音识别结果与 LLM 回复并排显示，被用户打断的回复会自动打上「已中止」标签 |
| 自动归档 | 挂断后整轮对话自动落入聊天历史，与文字对话共享一条时间线 |
| 状态机可视化 | 顶部状态点 + 文案明确指示「未连接 / 接通 / 监听中 / 思考中 / TTS 播报中」 |

> 与「唤醒词链路」共享同一套 Agent SDK，浏览器通话 = 手动发起，唤醒词 = ESP32 麦克风被动触发；两条链路同一份 LLM、同一份记忆、同一份技能。

### 1.3 技能（Skills）

![技能视图](img/30_skills.png)

四块横向并列，全部支持搜索 + 一键试播：

| 区块 | 说明 |
| --- | --- |
| **出厂技能** | 代码写死的核心能力（运动 / LED / 摄像头 / 记忆 / 反问），用户和 AI 都可调用，不可删 |
| **我的技能** | 由 OpenClaw / AI 现造的组合技能（串联若干出厂技能），支持卡片上 ✕ 直接删除，支持「重新加载」从磁盘热刷 |
| **录制动作** | 用户手动掰出来的轨迹；顶部「回放模式」胶囊一键切换 **平滑 / 原始 / 表现力** 三档；卡片直接点「播放」即可试 |
| **灯光表情** | 30+ LED 预设，点卡片直接下发 |

> "我的技能"是 Lampgo 区别于普通玩具机器人的关键 — **AI 帮你造的能力，会沉淀成可重复调用的卡片**。

### 1.4 OpenClaw（Agent 任务中枢）

![OpenClaw 视图](img/40_openclaw.png)

| 能力 | 说明 |
| --- | --- |
| 健康状态徽标 | 顶部一眼看到 OpenClaw 是否连通、插件是否注册成功 |
| 5 状态统计卡 | 排队中 / 进行中 / 待确认 / 已沉淀 / 失败 — 整体任务漏斗一屏看清 |
| 任务时间线 | 每个复杂任务的步骤、产物、所用工具、所用录制 / 技能均可展开 |
| 沉淀闭环 | "做一次"和"造成可重用技能"在同一视图完成，鼓励用户把成功的复杂任务保留下来 |

### 1.5 日志（Events）

![日志视图](img/50_events.png)

| 能力 | 说明 |
| --- | --- |
| 业务事件流 | 意图识别 / 技能触发 / 工具调用 / TTS 播报等用户能感知的事件 |
| 持久化 | 写入 `~/.lampgo/events.log`（JSONL，10MB × 3 轮转），刷新页面会回放最近 200 条，不会丢 |
| UI ≠ 终端 | UI 日志面向用户业务视角；终端 stderr 面向开发者（motor bus / LLM raw 请求等），两份不重叠 |

### 1.6 配置（Settings）

![配置 - 硬件](img/60_settings_hw.png)

四个 tab，覆盖全部可视化配置项：

| Tab | 内容 |
| --- | --- |
| **硬件** | 无线接入（ESP32 配网向导 + 设备列表 + 画面参数）、电机串口、摄像头/麦克风设备、运动高级参数（迪士尼动画十二法则的 Anticipation / Overlapping / Breathing）、安全限速 |
| **模型** | LLM Provider / Base URL / Key / 主模型 / 快速模型 / 消息格式 / 上下文窗口 / 温度；独立的 **MiMo 联网搜索** 子服务卡片；语音链路（STT / TTS / LiveKit / 火山引擎 / 静默超时）；Web 端口 |
| **人设** | SOUL / AGENTS / PROFILE 三份 Markdown 在线编辑，支持「从 OpenClaw 导入」和「恢复默认」 |
| **记忆** | 核心记忆（全量注入 prompt）+ 每日记忆（近 3 天自动归档注入），支持「立即总结当前会话」「从 OpenClaw 导入」 |

> 关键设计：**任何字段被 CLI 参数或环境变量覆盖时，会在该字段下方显式标灰提示来源**，配置覆盖优先级（CLI > config.toml > 默认值）完全透明。电机串口、摄像头和麦克风会热切换；设备标识、角度单位和 Web 端口这类启动期字段仍会提示冷重启。

## 2 · 演示剧本（6 个 MVP Case）

下面每个 Case 都按 **目标 → 操作步骤 → 现场要看的亮点** 三段组织，方便按页演示。

---

### Case 1 · 首次开机 5 分钟跑通

**目标**：从打开浏览器到台灯做出第一个动作，证明"开箱即用"。

**操作**：

1. 终端执行 `uv run lampgo run --web`，浏览器打开 `http://127.0.0.1:8420`。
2. 观察顶栏 4 个胶囊从灰色逐个变绿（关节 / 摄像头 / 麦克风 / LED）。
3. 在聊天首屏点击 hint chip **「做个点头」**。
4. 看回复区流式输出 + 工具调用气泡 + 真实台灯执行点头。

![Case 1 - 首屏](img/case1_hello.png)
![Case 1 - 工具调用气泡](img/case1_tool_call.png)

**亮点**：

- 顶栏胶囊 = 第一眼判断硬件是否齐活，不用看终端 log。
- 工具调用气泡可展开看参数 / 返回值 / 耗时，**对老板的 AI 透明度问题给到答案**。
- 聊天历史自动保存，不丢上下文。

---

### Case 2 · 手把手教一个新动作（录制 + 命名 + 自然语言播放）

**目标**：演示"AI + 用户共创动作库"的核心闭环。

**操作**：

1. 进入「技能」视图，找到「录制动作」区块，点击右上「录制动作」按钮。
2. 弹窗显示倒计时；点「开始录制」后电机自动松力矩，手动掰台灯做出"敬礼"。
3. 回到弹窗按回车结束，输入名字 `salute` 保存。
4. 录制卡片立即出现在区块内（**热加载，无需重启**），切换顶部「回放模式」为「平滑」试播。
5. 回到「新建聊天」输入「来个敬礼」— 看 LLM 自动路由到刚才的录制并播放。

![Case 2 - 录制对话框](img/case2_record_dialog.png)
![Case 2 - 录制卡片](img/case2_record_card.png)
![Case 2 - 自然语言播放](img/case2_play_via_chat.png)

**亮点**：

- **录制 / 命名 / 调用三步全部在前端完成**，不需要碰命令行或文件。
- 回放模式胶囊（平滑 / 原始 / 表现力）展示同一份轨迹能呈现完全不同的"性格"。
- "我的技能"区块同步显示由 AI 帮你封装的高阶技能，区分用户录制 vs AI 编排。

---

### Case 3 · "看看我桌面上有什么"（多模态视觉）

**目标**：证明 Lampgo 不只是一个动起来的灯，**它能看**。

**操作**：

1. 确保顶栏「摄像头」胶囊为绿色（本地或 ESP32 无线相机均可）。
2. 聊天框输入「帮我分析下桌面上有什么」。
3. 展开工具调用气泡里的 `camera_snap` — 可看到抓到的画面缩略图与传给 LLM 的 base64。
4. 回复中给出"桌面上有键盘、咖啡杯、笔记本…"等结构化描述。

![Case 3 - 视觉分析](img/case3_vision.png)

**亮点**：

- 视觉链路对用户完全透明：**LLM 决定要不要看 → 调 `camera_snap` → 拿到图 → 给出结论**。
- 切换顶栏摄像头胶囊即可在本地 USB 摄像头与 ESP32 无线相机之间秒切，演示"无线感知"的差异。

---

### Case 4 · 唤醒词全闭环 + 浏览器实时通话

**目标**：展示语音作为一等公民交互方式，两条链路统一回流。

**操作**：

1. 配置 → 模型 → 语音链路：确认 LiveKit / 火山引擎参数齐全、唤醒词为 `hey_jarvis`。
2. 远距离对着台灯说 **"Hey Jarvis，做个开心的表情"** — 听到提示音后说指令。
3. 切到左侧「通话」视图，可看到 ASR 文本与助手回复实时打字、表情同步变化。
4. 演示主动通话：点「通话」按钮 → 说话 → 中途打断 → 看到「已中止」标签 → 挂断后整通对话归档为聊天历史。

![Case 4 - 通话视图](img/case4_call_active.png)
![Case 4 - 被打断](img/case4_interrupted.png)

**亮点**：

- **被动唤醒**和**主动通话**共用一套 ASR / LLM / TTS / 记忆 / 技能，无人为切换成本。
- 打断时不仅 TTS 立即静音，**正在推理的 LLM 请求也会被取消**，避免长尾堆积。
- 静默超时自动挂断，省电、省 token、省尴尬。

---

### Case 6 · 让 AI 帮我造一个「欢迎回家」组合技能

**目标**：展示 **Agent 自我扩展能力** — 用户一句话，Lampgo 自己长出新技能。

**操作**：

1. 在聊天里说："帮我攒一个'欢迎回家'技能：先转头看门口、再眨眼、再播 welcome 录制。"
2. 切到「OpenClaw」视图，可看到任务从 **排队中 → 进行中 → 待确认**，逐步推进，每一步展开可看到调用了哪些工具。
3. 用户在「待确认」卡片上点「采纳」，任务进入 **已沉淀**。
4. 切到「技能」→「我的技能」，新卡片 `welcome_home` 已出现，描述、所需关节、所用录制全部列出。
5. 再回聊天输入「欢迎我回家」，AI 直接命中新技能并执行。

![Case 6 - OpenClaw 任务时间线](img/case6_openclaw_timeline.png)
![Case 6 - 沉淀后的技能卡片](img/case6_user_skill.png)

**亮点**：

- **能力可以被沉淀、可以被复用、可以被删除** — 不是一次性表演。
- OpenClaw 视图把"AI 在干什么 / 干到哪一步 / 我要不要采纳"三件事说清楚，老板能直接看到 ROI。
- 技能落地后是普通的 JSON + Markdown 文件，可走 Git 同步、可分享。

---

### Case 8 · ESP32 配网向导（开箱外设）

**目标**：演示无线感知设备的接入体验 — **对非工程师友好**。

**操作**：

1. 给 ESP32 上电；Lampgo 顶部出现黄色横幅「检测到未配网设备」。
2. 点横幅上的「配置 WiFi」按钮（或进入 设置 → 硬件 → 无线接入 → 配网）。
3. 走 4 步向导：
   - **Step 1** 引导用户连上 ESP32 临时热点 `Lampgo-Setup-XXXX`；
   - **Step 2** 选择 2.4GHz 家用 WiFi、输入密码（支持眼睛图标显示明文）；
   - **Step 3** 发送 → 设备重启 → 自动 mDNS 回连；
   - **Step 4** 完成，回到主界面，顶栏自动多出摄像头 / 麦克风 / 音量滑杆。

![Case 8 - 配网横幅](img/case8_banner.png)
![Case 8 - 配网向导](img/case8_wizard.png)
![Case 8 - 接入后顶栏](img/case8_topbar_after.png)

**亮点**：

- 无需命令行、无需 IDE、无需手动改 IP，**纯浏览器完成无线设备配网**。
- 失败兜底（找不到热点、长按 BOOT 重置、隐藏 SSID 手动输入）在向导里写明。
- 接入完成后立即可见画面 / 可调音量，**反馈即时**。
